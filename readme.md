校园流浪猫互助平台功能设计


一、架构
普通用户端：

校园流浪猫互助平台（普通用户端）	
├─ 0 启动页 & 微信快捷登录
├─ 1 核心首页（校园猫咪地图）
│  ├─ 猫咪/区域搜索功能
│  ├─ 地图热力分类（Tab切换：全部猫咪/需救助猫咪/已绝育猫咪）
│  └─ 快捷功能宫格（4个核心入口，关联高频操作）
├─ 2 流浪猫信息上报
├─ 3 救助需求申请
├─ 4 猫粮急救箱（资源互助）
├─ 5 交流大厅（师生互助社区）
├─ 6 猫咪档案管理
│  ├─ 档案列表（多维度筛选+排序）
│  └─ 猫咪详情（关注/投喂记录/领养申请）
├─ 7 个人中心（我的）
│  ├─ 我的申请记录（救助/领养/物资捐献）
│  ├─ 我的投喂记录
│  ├─ 我的关注猫咪
│  └─ 系统设置（缓存/反馈/关于）

二、页面功能

页码	页面名称	核心可见元素	用户操作流程	系统输出 / 反馈结果
0	启动页 & 登录	登录入口	输入手机号码以及密码登录
	登录成功后直接跳转至首页
1	首页	①顶部搜索栏（支持关键词搜索）
②校园地图组件
③底部 4 宫格按钮	点击宫格按钮跳转对应页面	按钮精准跳转至上报 / 申请等页面
2	流浪猫信息上报	①照片上传区
②地图选点（手动调整）
③提交按钮	①上传 1-9 张猫咪现场照片
②确认猫咪位置（可调）
③点击提交	生成页面提示 “信息已提交，待审核”。审核结果后续通过通知推送
3	救助需求申请	①猫咪选择（下拉选单 / 手动输地址）
②紧急度
③伤情材料区
④提交按钮	①选择需救助猫咪（或输入具体位置）
②选择紧急程度③上传伤情照片 + 填写描述；
④提交申请	生成救助工单号，实时推送至平台管理员端；用户端显示 “申请已受理，管理员将尽快响应”
4	猫粮急救箱		填写捐献信息	
5	交流大厅	“物资捐献”“救助申请”“投喂记录”板块。
	①　查看帖子，并进行评论、点赞等互动
②　发布帖子
查看投喂记录，进入“物资捐献”或“救助申请”板块	发布成功后显示 “帖子已发布”
6	猫咪档案	①顶部搜索栏（区域 / 健康 / 绝育 / 疫苗）
②猫咪档案卡片流（头像 + 昵称 + 状态 tag）	①点击卡片进入猫咪详情页	



管理员端
页码	页面名称	核心可见元素	用户操作流程	系统输出 / 反馈结果
0	管理员登录	登录入口、账号输入框、密码输入框	输入管理员账号及密码登录
	登录成功后跳转至控制台
1	控制台	①　审核中心模块
②　通知模块
③　档案模块
④　任务中心模块	点击宫格按钮跳转对应页面	按钮精准跳转至页面
2	审核中心	①　猫粮捐献审核模块
②　救助审核
③　领养审核	查看登记用户的申请，选择通过或驳回	审核结果推送至用户端
3	档案管理	猫咪信息	编辑档案信息	
4	通知模块	信息发布输入栏	填写站内信公告	推送至全站信箱
5	任务中心	任务发布模块	发布救助任务等	发布成功后显示 “任务已发布”
6	用户管理	用户列表，权限设置项	对用户进行权限管理（授权内部账号、封号）	处理结果推送用户端
